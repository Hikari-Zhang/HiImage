## 1. 产品概述

### 1.1 产品定位

HiImage 是一款 AI 驱动的图像处理桌面工具，主要面向需要批量去除水印、提升图像分辨率的个人用户和设计师。产品提供简洁直观的操作界面，通过深度学习模型实现高质量的图像修复与增强。

### 1.2 目标用户

| 用户群体 | 典型场景 |
|----------|----------|
| 个人用户 | 处理个人照片上的时间戳水印、平台水印 |
| 设计师 / 创作者 | 处理素材图片、提升低分辨率素材清晰度 |
| 内容运营 | 批量处理营销素材图片 |
| 开发者 | 自部署后端服务，集成到自有工作流 |

### 1.3 产品目标

- **易用性**: 三步完成水印去除（打开 → 选区 → 处理）
- **高质量**: 基于 15+ 个 AI 模型，覆盖各类场景
- **灵活部署**: 支持本地运行和远程 Docker 部署
- **可扩展**: 模块化架构，方便未来添加更多图像处理功能
- **稳定性**: 后端进程可靠管理，退出时干净清理资源

### 2.2 去水印 (Watermark Removal)

#### 功能描述

用户加载图片后，通过手动框选或自动检测标记水印区域，选择 AI 模型执行修复，获得去除水印后的图片。

#### 用户故事

- 作为用户，我希望能拖拽图片到应用中快速打开
- 作为用户，我希望能在图片上绘制矩形框标记水印位置
- 作为用户，我希望应用能自动检测图片中的水印区域
- 作为用户，我希望能选择不同的 AI 模型来处理不同类型的水印
- 作为用户，我希望看到处理前后的对比效果

#### 功能点

| # | 功能 | 优先级 | 描述 |
|---|------|--------|------|
| F1 | 图片加载 | P0 | 支持拖拽 / 文件对话框打开图片（PNG/JPG/BMP/WebP/TIFF） |
| F2 | ROI 绘制 | P0 | 鼠标拖拽在画布上绘制矩形选区，支持缩放/平移画布 |
| F3 | ROI 管理 | P0 | 列表展示所有选区，支持多选删除、清空所有 |
| F4 | 自动检测 | P1 | 基于 CV 算法自动识别水印位置，敏感度可调 |
| F5 | 模型选择 | P0 | 分组下拉选择模型（快速修复 / 专用模型 / 扩散模型） |
| F6 | 执行修复 | P0 | 点击按钮开始处理，显示实时进度条 |
| F7 | 结果预览 | P1 | 前后对比滑块 (Before/After) |
| F8 | 保存结果 | P0 | 通过系统对话框保存为 PNG/JPG |

#### 支持的模型

| 类别 | 模型 | 特点 |
|------|------|------|
| 快速修复 | LaMa | 推荐·通用，综合最佳 |
| 快速修复 | MiGAN | GAN 生成，速度快 |
| 快速修复 | ZITS | 边缘感知 |
| 快速修复 | MAT | 多尺度注意力，质量最高 |
| 快速修复 | CV2 | 传统算法，速度最快 |
| 专用模型 | AnyText | 文字水印专用 |
| 扩散模型 | SD Inpainting | 复杂背景 |
| 扩散模型 | Realistic Vision | 写实照片 |
| 扩散模型 | PowerPaintV2 | 文字引导，最强通用 |
| 扩散模型 | SDXL Inpainting | 高分辨率 |

### 2.5 设置 (Settings)

#### 功能描述

管理应用的全局配置，包括连接方式、计算设备、服务器参数、网络配置等。确保后端进程可靠启动、停止和切换。

#### 功能点

| # | 功能 | 优先级 | 描述 |
|---|------|--------|------|
| F21 | 连接方式切换 | P0 | 本地启动 / 远程连接 两种模式，切换时自动停止旧后端并启动新后端 |
| F22 | 远程配置 | P0 | 输入 IP + 端口，支持测试连接验证 |
| F23 | 计算设备 | P0 | MPS (Apple Silicon) / CPU / CUDA 选择 |
| F24 | 服务器配置 | P1 | IOPaint 端口、保活时间、启动超时 |
| F25 | 网络配置 | P1 | HuggingFace Endpoint 和 Token |
| F26 | NSFW 安全设置 | P2 | NSFW 内容过滤开关（待补充到设置界面） |
| F27 | 默认参数 | P2 | 默认遮罩扩张像素数 |
| F28 | 配置持久化 | P0 | 配置自动保存至 `config/settings.json`，重启后恢复 |
| F29 | 后端进程管理 | P0 | 退出应用时自动终止本地后端进程，无孤儿进程 |
| F30 | 端口冲突处理 | P0 | 启动本地后端前自动清理占用端口的残留进程 |

#### 设计细节

**连接模式切换行为（F21）**：
- 用户在"本地启动"和"远程连接"之间切换时，应用立即执行：
  1. 停止当前后端（本地进程终止 / 远程连接断开）
  2. 更新配置
  3. 启动新选择的后端
- 切换过程中显示 loading 动画，防止重复点击
- 切换成功后自动更新前端 `backendURL` store

**后端进程管理（F29）**：
- 使用串行队列（Promise chain）保证启动/停止操作按顺序执行，避免竞态条件
- 退出时通过 `forceQuit()` 跳过队列直接终止进程
- `before-quit` 事件处理：同步阻止默认行为，异步终止后端后强制退出
- 进程终止流程：`SIGTERM` → 等待 5 秒 → `SIGKILL`（强制）

**端口冲突处理（F30）**：
- 启动本地后端前，检查目标端口是否被占用
- 若被残留进程占用，自动执行 `kill -9` 清理
- 清理失败则提示用户手动处理

## 3. 非功能需求

### 3.1 性能

| 指标 | 要求 |
|------|------|
| 应用启动时间 | < 3s（不含模型加载） |
| 快速模型处理 | < 30s（1920x1080 图片） |
| UI 响应 | 处理过程中界面不卡顿（异步处理） |
| 内存占用 | 空闲时 < 200MB（不含模型） |

### 3.2 稳定性

| 指标 | 要求 |
|------|------|
| 后端进程清理 | 应用退出时 100% 清理本地后端进程 |
| 模式切换可靠性 | 快速连续点击不发生竞态错误 |
| 端口冲突恢复 | 自动清理残留进程，无需手动干预 |
| 崩溃恢复 | 异常退出后重启时能正常启动（无端口占用） |

### 3.3 兼容性

| 平台 | 支持情况 |
|------|----------|
| macOS (Apple Silicon) | 主要支持，MPS 加速 |
| macOS (Intel) | 支持，CPU 模式 |
| Windows | 计划支持（CUDA 加速） |
| Linux | Docker 部署支持 |

### 3.4 安全性

- HuggingFace Token 本地加密存储
- 后端仅监听 localhost（本地模式）
- Docker 部署时建议配合 VPN/内网使用

### 3.5 可用性

- 首次使用无需配置即可运行（默认 LaMa + MPS）
- 模型首次使用时自动下载，显示下载进度
- 操作失败时提供明确错误信息和建议

### 4.3 Backend Manager 设计

| 方法 | 说明 |
|------|------|
| `enqueue(task)` | 将任务加入串行队列，返回 Promise |
| `startLocalBackend()` | 启动本地后端：清理端口 → spawn 进程 → 健康检查 |
| `killLocalProcess()` | 终止进程：SIGTERM → 等待 → SIGKILL |
| `forceQuit()` | 退出时专用：abort 当前操作 → 重置队列 → kill |
| `freePort(port)` | 清理占用指定端口的残留进程 |
| `waitForReady(url)` | 轮询健康检查，支持 AbortController 取消 |
| `updateConfig()` | 更新配置：先 stop → 更新 → 根据新模式 start |

## 10. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 大模型下载慢 | 首次使用体验差 | 提供 HF Mirror 选项 + 下载进度显示 |
| MPS 内存不足 | 大图处理失败 | 自动降级到 CPU + 分块处理 |
| Docker GPU 兼容 | 特定显卡不支持 | 提供 CPU fallback 配置 |
| 模型更新 | 版本不兼容 | 模型版本锁定 + 自动下载指定版本 |
| 端口被占用 | 重启应用失败 | 启动前自动清理残留进程 |
| 竞态条件 | 快速操作导致状态错误 | 串行队列保证操作顺序 |

## 附录

### A. 术语表

| 术语 | 说明 |
|------|------|
| ROI | Region of Interest，用户标记的水印区域 |
| Inpainting | 图像修复，将水印区域重建为正常背景 |
| Super Resolution | 超分辨率，通过 AI 放大图像并增强细节 |
| IOPaint | 开源图像修复框架，支持多种模型 |
| Real-ESRGAN | 超分辨率模型，支持 2x/4x 放大 |
| MPS | Metal Performance Shaders，Apple 芯片 GPU 加速 |
| Dilation | 遮罩扩张，让遮罩略大于 ROI 确保完全覆盖水印边缘 |
| Serial Queue | 串行队列，保证异步操作按顺序执行，避免竞态条件 |
