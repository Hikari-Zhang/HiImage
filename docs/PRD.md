## 1. 产品概述

### 1.1 产品定位

HiImage 是一款 AI 驱动的图像处理桌面工具，面向设计师、内容创作者和个人用户。提供去水印、超分辨率、智能合成三大核心功能，基于 Electron + FastAPI 构建，支持本地和远程两种运行模式。

### 1.2 目标用户

| 用户群体 | 典型场景 |
|----------|----------|
| 个人用户 | 处理个人照片上的时间戳水印、平台水印 |
| 设计师 / 创作者 | 处理素材图片、提升低分辨率素材清晰度 |
| 内容运营 | 批量处理营销素材图片 |
| 开发者 | 自部署后端服务，集成到自有工作流 |

### 1.3 产品目标

- **易用性**：三步完成水印去除（打开 → 选区 → 处理）
- **高质量**：基于 15+ 个 AI 模型，覆盖各类场景
- **灵活部署**：支持本地运行和远程 Docker 部署
- **可扩展**：模块化架构，方便未来添加更多图像处理功能
- **稳定性**：后端进程可靠管理，退出时干净清理资源

### 2.2 去水印 (Watermark Removal)

#### 功能描述

用户加载图片后，通过手动框选或自动检测标记水印区域，选择 AI 模型执行修复，获得去除水印后的图片。

#### 功能点

| # | 功能 | 优先级 | 描述 |
|---|------|--------|------|
| F1 | 图片加载 | P0 | 支持拖拽 / 文件对话框打开图片（PNG/JPG/BMP/WebP/TIFF） |
| F2 | ROI 绘制 | P0 | 鼠标拖拽在画布上绘制矩形选区，支持缩放/平移画布 |
| F3 | ROI 管理 | P0 | 列表展示所有选区，支持多选删除、清空所有 |
| F4 | 自动检测 | P1 | 基于 CV 算法自动识别水印位置，敏感度可调 |
| F5 | 模型选择 | P0 | 分组下拉选择模型（快速修复 / 专用模型 / 扩散模型） |
| F6 | 执行修复 | P0 | 点击按钮开始处理，显示实时进度条 |
| F7 | 结果预览 | P1 | 前后对比滑块 (Before/After) |
| F8 | 保存结果 | P0 | 通过系统对话框保存为 PNG/JPG |

#### 支持的模型

| 类别 | 模型 | 特点 | 显存需求 |
|------|------|------|----------|
| 快速修复 | LaMa | 推荐·通用，综合最佳 | ~500MB |
| 快速修复 | MiGAN | GAN 生成，速度快 | ~1GB |
| 快速修复 | ZITS | 边缘感知 | ~1GB |
| 快速修复 | MAT | 多尺度注意力，质量最高 | ~2GB |
| 快速修复 | CV2 | 传统算法，速度最快 | 无 |
| 专用模型 | AnyText | 文字水印专用 | ~4GB |
| 扩散模型 | SD Inpainting | 复杂背景 | ~8GB |
| 扩散模型 | PowerPaint V1 | 文字引导，最强通用 | ~12GB |
| 扩散模型 | SDXL Inpainting | 高分辨率 | ~16GB |

| F12 | 执行超分 | P0 | 进度显示 + 异步处理 |
| F13 | 对比预览 | P1 | 拖拽滑块对比原图与超分结果 |
| F14 | 保存结果 | P0 | 同去水印模块 |

### 2.4 智能合成 (Smart Synthesis)

#### 功能描述

智能合成模块支持 **7 种合成模式**，覆盖从自动服装换色到自由指令编辑的广泛场景。

#### 模式总览

| 模式 | 名称 | 交互方式 | 后端技术 | 推荐场景 |
|---------|------|----------|----------|----------|
| `background_replace` | 换背景 | 上传背景图 | rembg 抠图 + 合成 | 商品图/人像换背景 |
| `outfit_swap` | 换装模拟 | 上传服装图 + 选 ROI | Inpainting 纹理替换 | 试穿上衣/裤子 |
| `face_swap` | 换脸模拟 | 上传人脸图 + 选 ROI | GFPGAN + 叠覆合成 | 人脸替换模拟 |
| `virtual_tryon` | 虚拟试穿 | 上传服装图 | 参考图引导 Inpainting | 在线试衣间 |
| `prompt_inpaint` | 精准替换 | 框选 ROI + 输入描述 | SD 1.5 Inpainting | 指定区域按描述重绘 |
| `auto_segment_edit` | 智能定位编辑 | 输入中文指令 | 自动分割 + 换色/换风格 | "将上衣换成红色" |
| `instruction_edit` | 自由编辑 | 输入任意指令 | InstructPix2Pix / FLUX | "把白天变成黄昏" |

#### 抠图模型（换背景用）

| 模型 | 特点 | 大小 |
|------|------|------|
| BiRefNet-General | 2024 SOTA，边缘极佳 | ~100MB |
| RMBG 2.0 | 商业级精度 | ~176MB |
| IS-Net General | 通用目标分割 | ~120MB |
| IS-Net Anime | 动漫/插画优化 | ~120MB |
| U²-Net | 经典显著目标检测 | ~176MB |
| MODNet | 轻量实时人像抠图 | ~25MB |

#### 合成模型（Inpainting）

| 模型 | 特点 | 大小 |
|------|------|------|
| FLUX.1-Fill-dev | 语义理解力极强，细节还原最佳 | ~23.8GB |
| SDXL | 1024px 高分辨率，纹理自然 | ~7GB |
| PowerPaint v2 | 多任务 Inpainting | ~5GB |
| LaMa | 大感受野卷积，纹理延续好 | ~200MB |
| SD 1.5 | 文生图引导，可用文字描述 | ~4GB |

#### 内存优化选项说明

| 选项 | 默认 | 作用 | 速度影响 |
|------|------|------|----------|

| Text Encoder CPU | 关闭 | Text Encoder 在 CPU 执行，省约 1GB 显存 | 极小 |

## 3. 非功能需求

### 3.1 性能

| 指标 | 要求 |
|------|------|
| 应用启动时间 | < 3s（不含模型加载） |
| 快速模型处理 | < 30s（1920x1080 图片） |
| UI 响应 | 处理过程中界面不卡顿（异步处理） |
| 内存占用 | 空闲时 < 200MB（不含模型） |

### 3.2 稳定性

| 指标 | 要求 |
|------|------|
| 后端进程清理 | 应用退出时 100% 清理本地后端进程及子进程 |
| 模式切换可靠性 | 快速连续点击不发生竞态错误 |
| 端口冲突恢复 | 自动清理残留进程，无需手动干预 |
| 崩溃恢复 | 异常退出后重启时能正常启动（无端口占用） |

### 3.3 兼容性

| 平台 | 支持情况 |
|------|----------|
| macOS (Apple Silicon) | 主要支持，MPS 加速 |
| macOS (Intel) | 支持，CPU 模式 |
| Windows | 支持（CUDA 加速） |
| Linux | Docker 部署支持 |

### 3.4 安全性

- HuggingFace Token 本地存储，不向外发送
- 本地模式后端仅监听 localhost
- Docker 部署时建议配合内网使用

```

│                    Electron App                          │
│  ┌──────────────────────────────────────────────────┐  │
│  │           React + TypeScript (Renderer)            │  │

         ┌──────────────▼──────────────┐

         │  ┌─────────┐ ┌───────────┐  │
         │  │Inpainter│ │Upscaler  │  │
```

## 8. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 大模型下载慢 | 首次使用体验差 | 提供 HF Mirror 选项 + 下载进度显示 |
| MPS 内存不足 | 大图处理失败 | 自动降级到 CPU + 分块处理 |
| Docker GPU 兼容 | 特定显卡不支持 | 提供 CPU fallback 配置 |
| 模型更新 | 版本不兼容 | 模型版本锁定（requirements-locked.txt）|
| 端口被占用 | 重启应用失败 | 启动前自动清理残留进程 |
| 竞态条件 | 快速操作导致状态错误 | 串行队列保证操作顺序 |
